# Nova ideia comercial - proposta Factorial

## Regra de ouro numero zero

- Sempre vender por **Bundle** primeiro.
- So considerar **Add-ons** quando houver gap funcional real.
- Evitar proposta com modulos soltos como primeira opcao.

## O que muda na proposta

1. Abrir pela dor do cliente:
   - app ruim no fornecedor atual;
   - falhas em clock-in remoto;
   - relatorios incompletos.
2. Recomendar um bundle como base.
3. Mostrar cobertura funcional do bundle em linguagem de resultado.
4. Mostrar Business vs Enterprise apenas no que impacta a decisao.
5. So no final, sugerir add-ons (se necessario).

## Linha comercial recomendada (sequencia)

1. **Diagnostico curto**: confirmar dor e prioridade.
2. **Escolha do bundle base**: Starter ou PRO.
3. **Validacao de cobertura**: Core + operacao + people needs.
4. **Gap analysis**: o que nao cobre no bundle.
5. **Add-ons cirurgicos**: apenas para fechar gaps.
6. **Implementacao simples**: etapas, prazo, responsaveis.
7. **Proxima acao clara**: proposta final + call curta.

## Pacote inicial para este caso (35 colaboradores)

- Bundle base recomendado: **Starter Planning** (ou equivalente validado na call).
- Cobertura principal: `Core + Time Tracking + Time Off + Shifts`.
- Add-on provavel: `Recruitment (5 Active Jobs)` se vagas ativas forem recorrentes.
- Add-ons futuros (somente se necessario): Engagement, Performance, Trainings.

## Mensagem pronta para proposta

"Para manter simplicidade de implantacao e controle de custo, seguimos o modelo Bundle-first: iniciamos com um bundle que cobre a operacao essencial de RH e adicionamos add-ons apenas se houver necessidade especifica de negocio."
